🏗️ Architecture Overview

Modular Design Pattern

Root Module (Entry Point):

Receives configuration from terraform.tfvars → Passes to 3 child modules → Aggregates outputs

Child Modules:

  • Queues: Creates genesyscloud_routing_queue resources
  • Users: Creates genesyscloud_user resources
  • Roles: Creates genesyscloud_auth_role resources

Data Flow:

terraform.tfvars → root/variables.tf → root/main.tf → module/variables.tf → module/main.tf → Resources in Genesys Cloud

🚀 Deployment Instructions

Prerequisites

  • Terraform >= v1.0.0
  • Genesys Cloud OAuth credentials
  • HCP Terraform account (for state management)

Step-by-Step Deployment

# Step 1: Initialize Terraform terraform init # Step 2: Validate configuration terraform validate # Step 3: Review changes terraform plan # Step 4: Apply configuration terraform apply # Step 5: View outputs terraform output

Environment Setup

Set these environment variables before deployment:

export GENESYSCLOUD_OAUTHCLIENT_ID="your-client-id" export GENESYSCLOUD_OAUTHCLIENT_SECRET="your-client-secret" export GENESYSCLOUD_REGION="your-region"

🔍 Troubleshooting Guide

Issue Root Cause Solution
Provider authentication fails Missing/invalid OAuth credentials Check GENESYSCLOUD_OAUTHCLIENT_ID and GENESYSCLOUD_OAUTHCLIENT_SECRET environment variables
Flow/Prompt not found Incorrect name or unavailable resource Verify exact names in Genesys Cloud or use direct IDs
Email validation error Invalid email format in terraform.tfvars Ensure emails follow name@domain.com format
State validation error User state not "active" or "inactive" Update terraform.tfvars with valid state values
Resource already exists Duplicate name in Genesys Cloud Check Genesys Cloud and update terraform.tfvars names
